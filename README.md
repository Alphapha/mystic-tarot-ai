# 🔮 Mystic Tarot AI · AI塔罗占卜大师

> 一手牌,一段命运。让 78 张塔罗牌 + 12 星座运势,在终端里为你照见此刻。
> 一款生活娱乐向的命令行占卜 Demo,神秘风 Rich UI,可选接入 AI 生成个性化解读。

---

## 一、作品介绍

**Mystic Tarot AI** 是一款把"塔罗占卜 + 星座运势"做成沉浸式命令行体验的生活娱乐 Demo。
没有花哨的网页,只有终端里跳动的紫金配色、缓缓翻开的牌面、与一句句温柔又有戏的解读——
就像深夜里坐在你对面那位话不多却看透一切的朋友。

它不是算命工具,而是一个**让普通人 30 秒就能获得"今日小仪式感"**的轻量娱乐应用:
早起抽一张牌作为今日指引,睡前抽三牌阵回看一天,顺便看看自己星座今天幸运色是什么。
所有结果都保存在本地,你可以回看自己的"命运时间线"。

### 为什么它能成爆款

- **强情绪价值**:占卜 + 星座是天然的情绪出口,年轻人每天通勤路上来一卦,仪式感拉满。
- **零门槛即玩**:纯 Python,装两个依赖就能跑,不联网也能用(本地解读降级方案)。
- **强可分享性**:终端截图自带高级感,朋友圈 / 小红书传播性极强。
- **可选 AI 加成**:接入大模型后,每张牌都变成"私人占卜师私语",体验直接升级一个档次。

## 二、功能说明

| # | 功能 | 说明 |
|---|------|------|
| 1 | 今日运势 | 输入星座或生日,生成 5 维度评分(整体/爱情/事业/财运/健康)+ 幸运色 / 数字 / 方位 + 寄语 |
| 2 | 每日一牌 | 抽一张塔罗牌作为今日指引,支持自定义问题 |
| 3 | 三牌阵占卜 | 经典「过去 - 现在 - 未来」牌阵,含综合解读 + 可选 AI 私语 |
| 4 | 查看历史 | 回看最近 20 条占卜记录(本地 `history.json`,最多保留 100 条) |
| 5 | 清空历史 | 一键清空所有记录 |
| 0 | 退出 | 关闭程序 |

### 核心特性

- **完整的 78 张塔罗牌库**:22 张大阿尔卡那(每张独立撰写正/逆位文案)+ 56 张小阿尔卡那(权杖/圣杯/宝剑/钱币四组)。
- **正逆位系统**:每张牌有 50% 概率逆位,含义随之反转,占卜结果千变万化。
- **跨进程稳定的每日运势**:同一星座同一天运势固定(基于 md5 种子),你和朋友看到的"狮子座今日运势"是一致的。
- **AI 解读(可选)**:配置 `.env` 后,调用 OpenAI 大模型为占卜结果生成 200 字内个性化解读;未配置时自动降级为本地解读,**不阻断主流程**。
- **隐私优先**:API Key 全部走 `.env`,代码内零硬编码;`history.json` 已加入 `.gitignore`。

## 四、安装与使用方法

### 环境要求

- Python 3.8+
- macOS / Linux / Windows 均可

### 安装步骤

```bash
# 1. 克隆仓库
git clone https://github.com/<你的用户名>/mystic-tarot-ai.git
cd mystic-tarot-ai

# 2. (推荐)创建虚拟环境
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# 3. 安装依赖
pip install -r requirements.txt
```

### 运行

```bash
python main.py
```

按菜单数字键选择功能即可,所有交互都在终端完成。

### 启用 AI 解读(可选,但强烈推荐)

```bash
# 1. 复制环境配置模板
cp .env.example .env

# 2. 编辑 .env,填入你的 OpenAI API Key
#    AI_ENABLED=true
#    OPENAI_API_KEY=sk-xxxxxxxxxxxxxxxx

# 3. 再次运行
python main.py
```

启动后顶部会显示 `[AI 解读已启用 · 模型 gpt-4o-mini]`,三牌阵占卜后会多出一段「AI 占卜师私语」。

> 兼容任何 OpenAI 协议的服务(Azure、各类中转、本地 vLLM 等),只需修改 `OPENAI_BASE_URL` 与 `OPENAI_MODEL`。

## 六、注意事项

1. **本应用纯属娱乐**:所有运势、占卜结果均为随机生成 + 文学化文案,不构成任何决策依据,请理性看待,切勿迷信。
2. **API Key 安全**:切勿将真实 `.env` 提交到 Git 仓库,`.gitignore` 已默认忽略。如不慎提交,请立即在 OpenAI 后台吊销 Key。
3. **AI 调用费用**:启用 AI 解读后会调用 OpenAI 接口,会产生少量 token 费用,默认模型 `gpt-4o-mini` 成本极低。
4. **网络要求**:本地解读模式无需联网;启用 AI 解读需能访问 `OPENAI_BASE_URL`(国内用户可配置中转地址)。
5. **历史记录**:`history.json` 保存在项目根目录,最多保留 100 条,可在菜单中选择「清空历史」一键清除。
6. **跨进程一致性**:同一星座同一天的运势是固定的(md5 种子),方便你和朋友对比;但塔罗抽牌每次都是随机的。

## 七、常见问题(FAQ)

**Q1:运行 `python main.py` 报错 `ModuleNotFoundError: No module named 'rich'`?**
A:依赖未安装。在项目目录执行 `pip install -r requirements.txt`。建议先激活虚拟环境。

**Q2:启动后显示「本地解读模式」,怎么开启 AI 解读?**
A:复制 `.env.example` 为 `.env`,把 `AI_ENABLED` 改为 `true` 并填入 `OPENAI_API_KEY`,再重新运行。

**Q3:AI 解读一直失败怎么办?**
A:程序会自动降级为本地解读,不影响使用。常见原因:API Key 无效、无法访问 OpenAI、模型名错误。可配置 `OPENAI_BASE_URL` 指向可达的中转服务,日志会打印具体错误。

**Q4:为什么我和朋友都是狮子座,但今天运势不完全一样?**
A:运势维度评分和幸运色是固定的(同一天同星座一致),但「今日寄语」从语料库随机抽取,会因调用时机不同而变化。如果想完全一致,可修改 `astrology.py` 让寄语也纳入种子。

**Q5:塔罗牌每次抽都不一样,正常吗?**
A:正常。塔罗抽牌使用真随机,每次都会重新洗牌,这是占卜应有的体验。

**Q6:为什么死神、塔这些牌看起来"不好"?**
A:塔罗中所谓"凶牌"往往象征转变与觉醒,正位未必坏、逆位未必好。本 Demo 的文案已尽量还原塔罗原意,建议结合自身处境感受,而非字面吉凶。

**Q7:`history.json` 会被提交到 GitHub 吗?**
A:不会,`.gitignore` 已忽略该文件。你的占卜记录完全保留在本地。

**Q8:支持中文乱码 / 牌面排版错乱?**
A:请使用支持 Unicode 的现代终端(macOS 自带 Terminal、iTerm2、Windows Terminal、VSCode 内置终端均可)。部分牌名含特殊符号,旧版 cmd 可能显示异常。

**Q9:可以做成网页版 / 小程序吗?**
A:当前 Demo 为 CLI 版本。核心引擎(`tarot_engine` / `astrology` / `ai_advisor`)与界面解耦,可直接被 Flask / FastAPI 包装为 Web API,欢迎二次开发。

**Q10:数据是真的占星学结论吗?**
A:不是。星座日期范围是公历近似值,运势为随机生成 + 文学化文案,仅供娱乐。请勿用于任何严肃决策。
